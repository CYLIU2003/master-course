# prepare-thesis-pr

## Goal
Explain a change in a way that is technically correct and defensible in a thesis or advisor review.

## Include
- motivation
- exact scope of change
- files changed
- logic or formula changes
- reproducibility impact
- risks and limitations
- tests / validation performed

## Output format
1. Motivation
2. Scope
3. Technical changes
4. Research impact
5. Validation
6. Remaining limitations
