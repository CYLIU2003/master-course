## このドキュメントの目的

本研究では、東急バスの BEV（電気バス）/ICE（エンジンバス）混成車両群を対象に、
**便割当・充電計画・電力調達を同時に最適化する MILP モデル** を実装しています。

4. 重要ファイルへの案内

## 1. 研究概要（3行で）

| 項目 | 内容 |
|------|------|
| **対象** | 東急バス全営業所の BEV/ICE 混成車両群 |

## 2. モデルの構造（全体像）

```
【入力データ】
  時刻表 / 車両諸元 / 電力料金（TOU・デマンド）/ PV 発電予測 / 契約電力上限
                            ↓
          ┌─────────────────────────────────────────────┐
          │           MILP 最適化モデル（Gurobi）          │
          │                                               │
          │  決定変数                                      │
          │  ・y_j^k   : 便 j を車両 k に割り当てるか      │
          │  ・c_{k,t} : 車両 k をスロット t に何 kW 充電  │
          │  ・g_t     : スロット t の系統買電量            │
          │                                               │
          │  制約（C1〜C21）                               │
          │  ・全便を必ず担当（欠便は大ペナルティ）          │
          │  ・走行中充電禁止 / SOC 上下限                 │
          │  ・充電器台数・出力の上限                       │
          │  ・系統受電量が契約電力以内                     │
          │                                               │
          │  目的関数（O1〜O4 + CO₂費 + 劣化費）           │
          └─────────────────────────────────────────────┘
                            ↓
【出力】
  車両ごとの運行スケジュール / 充電計画 / 費用内訳 / 未充足便リスト
```

## 3. 目的関数の説明

### 3.1 「O1 と O2 をなぜ一緒に最小化するか」（設計上の重要な前提）

> **O1（ICE 燃料費）と O2（電気代）は、1 本の式 $C_{total} = O1 + O2 + O3 + O4$ として同時に最小化します。**
>
> - O1 だけを最小化すれば → 「ICE バスを 1 台も使わない」が自明な最適解となり、研究上の意味がありません。
> - O2 だけを最小化すれば → 「BEV を 1 台も充電しない（ICE だけ使う）」が自明な最適解となり、同様に無意味です。
>
> **O1 と O2 を 1 本の式で足し合わせて最小化することで、初めてトレードオフが生まれます。**
> 「ICE を使えば燃料費（O1）が増え、BEV を充電すれば電気代（O2）が増える」という構造になり、
> ソルバーは ICE と BEV の **最適な混合比率・充電タイミング** を自動で決定します。

### 3.2 最小化する式（全体）

$$
\min \quad C_{total} = \underbrace{O1 + O2 + O3 + O4}_{\text{常に有効}} + \underbrace{\sum_j \pi \cdot u_j}_{\text{欠便ペナルティ（常に有効）}} + \underbrace{C_{CO_2}^{*} + C_{degr}^{*}}_{\text{パラメータ設定時のみ有効}}
$$

各項の展開：

$$
O1 = c_f \sum_{k \in \text{ICE}} \bigl( \sum_j f_k(j) y_j^k + \sum_{(i,j)} f_k^{dh}(i,j) x_{ij}^k \bigr), \quad
O2 = \sum_t p_t^{grid} \cdot g_t
$$

$$
O3 = p^{dem,on} W^{on} + p^{dem,off} W^{off}, \quad
O4 = \sum_k c_k^{veh} z_k
$$

$$
C_{CO_2}^{*} = p^{CO_2} \Bigl( \alpha_{ICE} \sum_{k \in \text{ICE},j} f_k(j) y_j^k + \alpha_{grid} \sum_t g_t \Bigr) \quad (p^{CO_2} > 0\text{ のとき有効})
$$

$$
C_{degr}^{*} = w^{degr} \sum_{k \in \text{BEV},t} \frac{c_{k,t} \cdot \Delta t}{cap_k} \beta \quad (w^{degr} > 0\text{ のとき有効})
$$

#### 各項の有効化条件

| 費目 | 有効化条件 | 備考 |
|------|-----------|------|
| O1 ICE 燃料費 | **常に有効** | ICE 車両が 0 台なら値が 0 になるだけ |
| O2 電気代（TOU） | **常に有効** | 系統買電量が 0 なら値が 0 になるだけ |
| O3 デマンド料金 | **常に有効** | 単価パラメータを 0 にすれば実質無効化可 |
| O4 車両固定費 | **常に有効** | 固定費パラメータを 0 にすれば実質無効化可 |
| 欠便ペナルティ | **常に有効・無条件** | コード上ですべての費目の後に無条件加算。最低 10,000 円/便 |
| CO₂ 費用 | `co2_price_per_kg > 0` のときのみ | デフォルト 0（無効） |
| 電池劣化費 | `degradation_weight > 0` のときのみ | デフォルト 0（無効） |

#### 変数の定義

**決定変数（ソルバーが値を決める）：**

| 記号 | 意味 | 型・範囲 |
|------|------|---------|
| $y_j^k$ | 車両 $k$ が便 $j$ を担当するか | 0/1 整数 |
| $x_{ij}^k$ | 車両 $k$ が便 $i$ → $j$ の回送接続をするか | 0/1 整数 |
| $u_j$ | 便 $j$ が担当不能（欠便）か | 0/1 整数 |
| $z_k$ | 車両 $k$ が 1 日に使われるか | 0/1 整数 |
| $c_{k,t}$ | 車両 $k$ のスロット $t$ での充電電力 | 連続 $[0, c_{\max,k}]$（kW） |
| $g_t$ | スロット $t$ での系統買電量 | 連続 $\geq 0$（kWh） |
| $W^{on}$ | オンピーク期間の最大需要電力 | 連続 $\geq 0$（kW） |
| $W^{off}$ | オフピーク期間の最大需要電力 | 連続 $\geq 0$（kW） |

**パラメータ（入力として与えられる定数）：**

| 記号 | 意味 | 単位 |
|------|------|------|
| $c_f$ | 軽油単価 | 円/L |
| $f_k(j)$ | 便 $j$ の燃料消費量 | L |
| $f_k^{dh}(i,j)$ | 回送走行の燃料消費量 | L |
| $p_t^{grid}$ | スロット $t$ の電力単価（TOU） | 円/kWh |
| $p^{dem,on/off}$ | オン・オフピークのデマンド単価 | 円/kW |
| $c_k^{veh}$ | 車両 $k$ の日割り固定費 | 円/日 |
| $\pi$ | 欠便ペナルティ単価（最低 10,000） | 円/便 |
| $p^{CO_2}$ | CO₂ 価格（デフォルト 0） | 円/kg |
| $\alpha_{ICE}$ | 軽油の CO₂ 排出係数（既定 2.64） | kg/L |
| $\alpha_{grid}$ | 系統電力の CO₂ 排出係数 | kg/kWh |
| $w^{degr}$ | 劣化費の重み（デフォルト 0） | 無次元 |
| $\beta$ | 1 サイクル当たり劣化コスト（現在 50.0 固定） | 円/cycle |
| $cap_k$ | バッテリー容量（既定 300） | kWh |
| $\Delta t$ | 1 時間スロットの幅 | h |

### 3.3 各費目の直感的な意味

| 費目 | なぜ最小化に含めるか | 組み合わせることの意義 |
|------|-------------------|--------------------|
| **O1 ICE 燃料費** | ICE バスの走行コスト | O2 と合わせることで「ICE ↔ BEV の最適混合比率」が決まる |
| **O2 電気代（TOU）** | 充電時刻をずらすと安い時間帯に充電でき、コストが下がる | O1 と合わせることで「充電量を無制限に増やすのではなく必要量だけ充電する」誘因が働く |
| **O3 デマンド料金** | 月の最大需要電力 × 単価が基本料金となる。充電を分散させると削減できる | ピーク充電の抑制・分散化を誘導する |
| **O4 固定費** | 使用台数が多いほど高くなる日割り固定費 | 必要最小台数で運用する方向に働く |
| **欠便ペナルティ** | 担当不能な便を実質的に禁止するための大きな罰則（通常 10,000 円/便） | — |
| **CO₂費（オプション）** | 排出量に価格を付け、CO₂ の少ない車種・充電時間帯を誘導する | `co2_price_per_kg > 0` に設定すると有効化 |
| **劣化費（オプション）** | 充電量が多いほど電池が劣化する。過充電を抑制する | `degradation_weight > 0` に設定すると有効化 |

### 3.4 デフォルト設定について

CO₂費・劣化費は `co2_price_per_kg = 0`（無効）がデフォルトです。
パラメータに正の値を設定すると目的関数に加算されます。
これにより、単純コスト最小化 / CO₂考慮 / 劣化考慮 を切り替えられます。

## 8. 用語対応表

数理最適化に慣れていない方向けに、用語を整理します。

| 数理用語 | 直感的な意味 |
|---------|------------|
| 目的関数 | モデルが最終的に一番小さくしたい「合計費用の計算式」 |
| 制約 | 現実の運行で絶対に破ってはいけない条件 |
| 決定変数 | モデルが自動で決める項目（便割当・充電量など） |
| MILP | 整数変数（0 or 1）と連続変数が混在する線形最適化問題の総称 |
| SOC | バッテリー残量（State of Charge）。ここでは kWh 単位 |
| TOU | 時間帯別料金（Time of Use）。電気代が時間帯によって異なる料金体系 |
| デマンド料金 | 月の最大需要電力（ピーク電力）に応じた基本料金 |
| ALNS | 適応型大規模近傍探索（近似解法。大規模問題向け） |
| deadhead | バスが乗客なしで移動すること（回送） |
| feasible connection | 時刻的に接続可能な便の組み合わせ |
