# AGENTS.md
## シミュレーションツールへ「エンジンバス性能指標」を取り込むためのエージェント指示書

## ステータス
- 分類: 参考資料
- 用途: JH25 ベースのエンジンバス性能指標取り込み手順を確認する
- 備考: エンジンバス統合の個別作業時に参照

### 1. 目的
このエージェントの目的は、アップロード済みの JH25 モード燃費データ Excel を参照し、**エンジンバス（ディーゼルバス）性能指標**を抽出・標準化し、既存のシミュレーションツールへ安全に組み込める形に変換することである。

本エージェントは、BEV バスのみを前提にしたモデルを、**エンジンバスも比較対象として扱える混成評価モデル**へ拡張するための補助を行う。

---

### 2. 参照元ファイル
以下の Excel ファイルを一次資料として扱うこと。

- `/mnt/data/bus_isuzu_jh25.xlsx`
- `/mnt/data/bus_hino_jh25.xlsx`
- `/mnt/data/mitsubishifuso_bus_jh25.xlsx`

上記ファイルはいずれも、JH25 モードにおけるバスの燃費・CO2 排出量・車両諸元を含む。
基本的には各ファイルの `3-1` シートを読むこと。

---

### 3. エージェントの役割
エージェントは次を行うこと。

1. Excel からエンジンバス関連行を抽出する  
2. 列名ゆれ・表記ゆれを吸収して標準化する  
3. シミュレーションで使える共通パラメータへ変換する  
4. 比較対象として使いやすいように「代表値」「候補群」「保守値」を整理する  
5. 出力を Markdown / JSON / CSV のいずれかで再利用可能な形に整える  

---

### 4. 本タスクで最低限抽出すべき指標
以下の項目を**必須項目**として扱うこと。

#### 4.1 車両識別系
- manufacturer : メーカー名  
- vehicle_name : 車名 / 通称名  
- model_code : 型式  
- bus_category : `路線バス` または `一般バス`  
- source_file : 参照元 Excel ファイル名  
- source_sheet : 参照元シート名  
- source_row : 抽出元行番号  

#### 4.2 パワートレイン系
- engine_model : 原動機型式  
- displacement_L : 総排気量 [L]  
- max_torque_Nm : 最大トルク [N-m]  
- max_power_kW : 最高出力 [kW]  
- transmission : 変速装置の型式及び変速段数  

#### 4.3 車両質量・容量系
- vehicle_weight_kg : 車両重量 [kg]  
- gross_vehicle_weight_kg : 車両総重量 [kg]  
- passenger_capacity : 乗車定員 [名] または最大積載量  
- structure_type : 自動車の構造  
- wheel_arrangement : 車輪配列  

#### 4.4 環境・効率系
- fuel_economy_km_per_L : JH25 モード燃費値 [km/L]  
- co2_g_per_km : 1km 走行あたり CO2 排出量 [g-CO2/km]  
- fuel_standard_km_per_L : 令和7年度燃費基準値 [km/L]  
- fuel_standard_achievement_level : 燃費基準達成レベル [%] 相当  
- low_emission_certification : 低排出ガス認定レベル  
- fuel_saving_measures : 主要燃費改善対策  
- emission_measures : 主要排出ガス対策  
- notes : その他燃費値の異なる要因・備考

### 5. シミュレーション用に追加で作る派生指標
シミュレーション内で使いやすくするため、以下の派生値を計算すること。

#### 5.1 燃料消費量
- `diesel_consumption_L_per_km = 1 / fuel_economy_km_per_L`

#### 5.2 100km あたり燃料消費量
- `diesel_consumption_L_per_100km = 100 / fuel_economy_km_per_L`

#### 5.3 1人あたり CO2 指標（定員ベース、任意）
- `co2_g_per_pax_km = co2_g_per_km / passenger_capacity`
- passenger_capacity が欠損・0 の場合は計算しないこと

#### 5.4 1人あたり燃料消費指標（任意）
- `diesel_L_per_pax_km = diesel_consumption_L_per_km / passenger_capacity`

#### 5.5 シミュレーション用の便宜的エネルギー換算値（任意）
ツール側がエネルギー比較を必要とする場合のみ、次を使ってよい。

- `diesel_energy_kWh_per_L = 9.8` を初期既定値とする  
- `equivalent_energy_kWh_per_km = diesel_consumption_L_per_km * diesel_energy_kWh_per_L`

注意:
- これは**燃料の低位発熱量ベース等の便宜換算**であり、BEV の電費と直接同義ではない  
- 厳密比較が必要な場合は別途、駆動系効率や実走行補正係数を導入すること

### 6. データ抽出ルール
#### 6.1 ヘッダ検出
元 Excel は多段ヘッダになっているため、固定列番号だけに依存しないこと。  
ただし今回の参照ファイルでは `3-1` シートの表構造がほぼ共通なので、以下の意味列を検出対象とする。

- 車名
- 通称名
- 型式
- 原動機型式
- 総排気量(L)
- 最大トルク(N-m)
- 最高出力(kW)
- 変速装置の型式及び変速段数
- 車両重量(kg)
- 車両総重量(kg)
- 最大積載量(kg)又は乗車定員(名)
- 自動車の構造
- 燃費値(km/L)
- 1km走行におけるCO2排出量(g-CO2/km)
- 令和7年度燃費基準値(km/L)
- 主要燃費改善対策
- 主要排出ガス対策
- 車輪配列
- その他

#### 6.2 欠損時の扱い
- 前行と同一車名・同一シリーズでセルが空欄の場合、**結合セル由来の省略**の可能性を考慮すること
- ただし、安易に埋めず、**明らかに同一グループである場合のみ前方補完**すること
- 補完した項目は `imputed_fields` に記録すること

#### 6.3 数値変換
- 数値列は必ず numeric に変換すること
- 変換不能値は null とし、元文字列を `raw_text_backup` に残してよい

### 7. シミュレーション統合時の基本方針
#### 7.1 エンジンバスの役割
エンジンバスは、少なくとも次の 3 つの用途で扱えるようにすること。

1. **BEV との比較対象**
   - CO2
   - エネルギー消費
   - 運行可能距離制約の違い
   - 車両性能差

2. **混成運用の構成要素**
   - 一部便を BEV
   - 一部便を エンジンバス
   - コストや CO2 を同時比較

3. **導入移行期のベースライン**
   - 全便エンジンバス
   - 一部 BEV 化
   - 全便 BEV 化
   の3ケース比較

#### 7.2 シミュレーションに最低限渡すべき共通パラメータ
既存ツール側へは少なくとも次の形で渡すこと。

```json
{
  "vehicle_id": "hino_blue_ribbon_01",
  "vehicle_type": "engine_bus",
  "manufacturer": "Hino",
  "model_code": "2KG-XXXX",
  "bus_category": "route_bus",
  "passenger_capacity": 79,
  "gross_vehicle_weight_kg": 14548,
  "fuel_economy_km_per_L": 4.38,
  "diesel_consumption_L_per_km": 0.2283,
  "co2_g_per_km": 590.46,
  "max_power_kW": 199,
  "max_torque_Nm": 785,
  "transmission": "6AT",
  "source": {
    "file": "bus_hino_jh25.xlsx",
    "sheet": "3-1",
    "row": 123
  }
}
```

### 8. 車種選定ロジック
同一メーカー内に複数の仕様があるため、用途に応じて次の 3 種類の選び方を実装すること。

#### 8.1 exact_match
ユーザが以下を指定した場合は、できる限り完全一致で選ぶ。

- メーカー
- 路線バス / 一般バス
- 定員
- 型式
- 車両総重量帯
- 出力帯

#### 8.2 representative
一致候補が複数ある場合、**代表値モデル**を返す。
原則は以下。

- 路線バスなら、路線バスの中で標準的な定員・総重量のもの
- 一般バスなら、一般バスの中で標準的な定員・総重量のもの
- 同一系列で燃費違いがある場合は中央値に近いもの

#### 8.3 conservative
安全側・悪化側評価が必要な場合は以下を返す。

- CO2 は高め
- 燃費は低め
- エネルギー消費は大きめ

これは、導入効果を過大評価しないために使う。

### 9. 推奨データセット構造
エージェントは最終的に、以下の 3 層を作れるようにすること。

#### 9.1 raw_records
Excel から読んだ元データに近いレコード群

#### 9.2 normalized_records
列名・単位・表記ゆれを統一したレコード群

#### 9.3 simulation_vehicle_library
シミュレーション投入用に絞り込んだ車種ライブラリ

例:
- `route_bus_small_engine`
- `route_bus_standard_engine`
- `route_bus_large_engine`
- `coach_standard_engine`

---

### 10. 代表車種のまとめ方のルール
現時点では、各メーカーから最低 1 台以上、できれば以下の区分で代表候補を作ること。

- 小型寄り一般バス
- 標準的路線バス
- 大型一般バス / 観光・高速系寄り
- 高出力帯モデル

同系列で類似型式が多数ある場合は、全部をそのまま投入せず、まずは**代表車種セット**を別途作ること。

---

### 11. 品質チェック
出力前に以下を必ず確認すること。

- `fuel_economy_km_per_L > 0`
- `co2_g_per_km > 0`
- `diesel_consumption_L_per_km = 1 / fuel_economy_km_per_L` が成立しているか
- 同一型式なのに数値が大きく異なる場合、重量差や仕様差が説明可能か
- `bus_category` が `路線バス` / `一般バス` のいずれかに整理されているか
- null が多いレコードは `needs_manual_review = true` を付けること

---

### 12. 出力形式の優先順位
出力形式は次の順で優先すること。

1. `JSON`
   - シミュレーションツール取り込み用
2. `CSV`
   - 一括確認・表計算確認用
3. `Markdown`
   - 人間向け確認用サマリ
4. `Python dict/list`
   - プロトタイプ実装用

---

### 13. Markdown サマリの出し方
Markdown で人間向けに整理する場合は、最低でも以下の表を出力すること。

| manufacturer | vehicle_name | model_code | bus_category | capacity | GVW_kg | fuel_km_per_L | CO2_g_per_km | power_kW | torque_Nm |
|---|---|---|---|---:|---:|---:|---:|---:|---:|

さらに、必要に応じて以下も出すこと。

- メーカー別の燃費レンジ
- バス区分別の CO2 レンジ
- 代表車種候補一覧
- 要手確認レコード一覧

### 14. 今回の参照ファイルから把握しておくべき傾向
今回のアップロード済みファイル群には、少なくとも以下のような傾向が含まれている。

- いすゞ:
  - ガーラ系の一般バスが含まれる
  - 燃費はおおむね 4 km/L 台後半 - 5 km/L 台後半の帯が見られる
  - CO2 はおおむね 450 - 580 g/km 前後の帯が見られる

- 日野:
  - リエッセ II など小型寄り一般バスが含まれる
  - 9 km/L 台の高燃費な小型系が見られる
  - 路線バス系では 4 - 5 km/L 台程度の帯が想定される

- 三菱ふそう:
  - エアロエース / エアロクィーン / エアロスターが含まれる
  - 一般バス・路線バスの両方が見られる
  - 5 km/L 前後帯の大型車が含まれる

これらは**初期代表車種ライブラリ作成の参考**に使ってよい。

### 15. してはいけないこと
- BEV の電費値とディーゼルの km/L を、そのまま同列比較しないこと
- 元 Excel の空欄を、根拠なく勝手に埋めないこと
- 型式違い・重量違いの車両を同一個体として混同しないこと
- 参照元行番号を消さないこと
- 単位を落としたままシミュレーションへ渡さないこと

### 18. 最終目的
最終的にはこのエージェントにより、シミュレーションツール内で次を比較可能にする。

- エンジンバス単独運用
- BEV バス単独運用
- エンジンバス + BEV の混成運用
- PV あり / なし
- CO2 / エネルギー / 運行成立性 / コスト

以上を前提として、**元 Excel の JH25 燃費・CO2 データを、比較可能な標準化車両ライブラリへ変換すること**をこのエージェントの中核任務とする。
