## 0. 前提知識：ODPTデータの構造（JSON-LD）

ODPTはJSON-LD（JSON for Linking Data）形式を採用している。全てのオブジェクトに以下のフィールドが存在する。

```
@context     → JSON-LDのコンテキスト定義（名前空間マッピング）
@type        → オブジェクト種別（例: "odpt:BusTimetable"）
@id          → オブジェクトのURI（内部表現）
owl:sameAs   → 一意識別子URI ← これが主キー相当
dc:date      → 更新日時（ISO 8601）
dct:valid    → 有効期限（ISO 8601）← キャッシュ判定に使う
```

**名前空間プレフィックス一覧**（列名変換時に必ず除去すること）:

| プレフィックス | 意味 | 例 |
|-------------|------|-----|
| `odpt:` | ODPT固有フィールド | `odpt:operator`, `odpt:busroute` |
| `dc:` | Dublin Core | `dc:date`, `dc:title` |
| `dct:` | Dublin Core Terms | `dct:valid` |
| `geo:` | WGS84 地理座標 | `geo:lat`, `geo:long` |
| `owl:` | Web Ontology Language | `owl:sameAs` |

**バス関連の主要オブジェクト型**:

```
odpt:BusTimetable        → 時刻表ヘッダ（路線・系統・暦ごと）
odpt:BusroutePattern     → 系統パターン（停留所順序・距離）
odpt:BusstopPole         → 停留所ポール（位置・名称）
odpt:Bus                 → 車両リアルタイム位置
odpt:Calendar            → 運行カレンダー（平日/土/日祝）
```

## 1. odpt/cache.py の完全実装

キャッシュ機構は全てのfetch処理の前に必ず呼び出す。

```python
# odpt/cache.py
"""
ODPT APIキャッシュ管理モジュール。
dct:valid フィールドに基づくキャッシュ有効期限チェックと
バージョン管理付きファイル保管を担う。

【報告書 3.6節 APIキャッシュ戦略 より】
- 静的データ（BusTimetable, BusroutePattern, BusstopPole）:
  ダイヤ改正時のみ再取得（数ヶ月〜年単位）
- 動的データ（Bus車両位置）:
  dct:valid を確認し、期限内はキャッシュを利用
"""

import json
from datetime import datetime, timezone
from pathlib import Path
from typing import Optional


def is_cache_valid(obj: dict) -> bool:
    """
    単一のODPTオブジェクトのdct:validを確認し、有効期限内かを返す。
    dct:validが存在しない場合はFalseを返す（再取得を促す）。
    """
    valid_str = obj.get("dct:valid")
    if not valid_str:
        return False
    try:
        # ISO 8601形式: "2025-04-01T00:00:00+09:00"
        valid_dt = datetime.fromisoformat(valid_str)
        # タイムゾーン情報がある場合はaware比較、ない場合はnaive比較
        now = datetime.now(valid_dt.tzinfo) if valid_dt.tzinfo else datetime.now()
        return now < valid_dt
    except ValueError:
        return False


def find_latest_cache(data_dir: str, endpoint_name: str) -> Optional[Path]:
    """
    data/raw/ から最新タイムスタンプのJSONキャッシュファイルを探す。
    例: bustimetable_20250301.json → 返す
    存在しない場合は None を返す。
    """
    raw_dir = Path(data_dir)
    pattern = f"{endpoint_name.lower()}_*.json"
    candidates = sorted(raw_dir.glob(pattern), reverse=True)  # 降順=最新が先頭
    return candidates[0] if candidates else None


def load_cache_or_none(data_dir: str, endpoint_name: str) -> Optional[list]:
    """
    キャッシュファイルを読み込み、先頭オブジェクトのdct:validが
    有効であればデータリストを返す。無効または存在しない場合はNoneを返す。
    """
    cache_path = find_latest_cache(data_dir, endpoint_name)
    if cache_path is None:
        print(f"[cache] {endpoint_name}: キャッシュなし → API取得が必要")
        return None

    with open(cache_path, encoding="utf-8") as f:
        data = json.load(f)

    if not data:
        return None

    # 先頭オブジェクトのdct:validで代表チェック（全件チェックは不要）
    if is_cache_valid(data[0]):
        print(f"[cache] {endpoint_name}: キャッシュ有効 ({cache_path.name})")
        return data
    else:
        print(f"[cache] {endpoint_name}: キャッシュ期限切れ → API取得が必要")
        return None


def save_cache(data: list, data_dir: str, endpoint_name: str) -> Path:
    """
    取得したデータをタイムスタンプ付きJSONファイルとして保存する。
    """
    raw_dir = Path(data_dir)
    raw_dir.mkdir(parents=True, exist_ok=True)
    date_str = datetime.now().strftime("%Y%m%d")
    out_path = raw_dir / f"{endpoint_name.lower()}_{date_str}.json"
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    print(f"[cache] 保存: {out_path} ({len(data)}件)")
    return out_path


def record_db_version(db_path: str, endpoint_name: str, source_file: str):
    """
    SQLiteのversionテーブルに最終更新日時を記録する。
    ダイヤ改正管理に使用。
    """
    import sqlite3
    con = sqlite3.connect(db_path)
    con.execute("""
        CREATE TABLE IF NOT EXISTS _data_version (
            endpoint TEXT PRIMARY KEY,
            source_file TEXT,
            updated_at TEXT
        )
    """)
    con.execute("""
        INSERT OR REPLACE INTO _data_version(endpoint, source_file, updated_at)
        VALUES (?, ?, ?)
    """, (endpoint_name, source_file, datetime.now(timezone.utc).isoformat()))
    con.commit()
    con.close()
```

## 2. odpt/fetch.py の完全実装

```python
# odpt/fetch.py
"""
ODPT API v4 からバス関連データを取得する。
キャッシュが有効な場合はAPI呼び出しをスキップする。

【対象エンドポイント】
- BusTimetable    : 時刻表（路線×系統×暦の組み合わせ）
- BusroutePattern : 系統パターン（停留所順序）
- BusstopPole     : 停留所マスタ
- Bus             : 車両リアルタイム位置（動的データ）

【目黒営業所対象路線】
黒01, 黒02, 東98, 渋71, 渋72, さんまバス（季節運行）
"""

import os
import time
import requests
from pathlib import Path
from dotenv import load_dotenv

from odpt.cache import load_cache_or_none, save_cache

load_dotenv()  # .env から ODPT_API_KEY を読み込む

ODPT_BASE = "https://api.odpt.org/api/4/"
RAW_DIR = "data/raw"

# ---- 事業者IDの注意 ----
# 東急バスは "odpt.Operator:Tokyu" ではなく正確なIDを確認すること。
# ODPT APIで以下のクエリを一度実行して確認する:
#   GET /api/4/objects?@type=odpt:Operator&acl:consumerKey=YOUR_KEY
# 確認後、以下の定数を更新すること。
TOKYU_OPERATOR = "odpt.Operator:TokyuBus"  # ← 要確認・修正

# 目黒営業所が担当する路線の busroute ID（ODPTの owl:sameAs 形式）
# fetch後に確認・修正すること。
MEGURO_ROUTES = [
    "odpt.Busroute:TokyuBus.Kuro01",   # 黒01（目黒駅〜中目黒駅前）
    "odpt.Busroute:TokyuBus.Kuro02",   # 黒02
    "odpt.Busroute:TokyuBus.Higashi98", # 東98
    "odpt.Busroute:TokyuBus.Shibu71",  # 渋71
    "odpt.Busroute:TokyuBus.Shibu72",  # 渋72
]


def get_api_key() -> str:
    key = os.environ.get("ODPT_API_KEY")
    if not key:
        raise EnvironmentError(
            "環境変数 ODPT_API_KEY が設定されていません。"
            ".env ファイルに ODPT_API_KEY=your_key_here を追加してください。"
        )
    return key


def fetch_endpoint(api_key: str, odpt_type: str, extra_params: dict = None) -> list:
    """
    指定エンドポイントから全件取得する。
    APIはページネーションなし（一括返却）。
    """
    params = {
        "acl:consumerKey": api_key,
        "@type": odpt_type,
    }
    if extra_params:
        params.update(extra_params)

    try:
        resp = requests.get(
            ODPT_BASE + "objects/",
            params=params,
            timeout=60,
        )
        resp.raise_for_status()
        return resp.json()
    except requests.exceptions.Timeout:
        print(f"[fetch] タイムアウト: {odpt_type}。リトライしてください。")
        raise
    except requests.exceptions.HTTPError as e:
        print(f"[fetch] HTTPエラー {e.response.status_code}: {odpt_type}")
        raise


def fetch_all_static(force_refresh: bool = False):
    """
    静的マスタデータ（時刻表・系統・停留所）を一括取得・保存する。
    キャッシュが有効な場合はスキップ（force_refresh=True で強制再取得）。
    ダイヤ改正後に1回だけ実行する。
    """
    api_key = get_api_key()

    STATIC_ENDPOINTS = {
        "BusTimetable": {
            "@type": "odpt:BusTimetable",
            "odpt:operator": TOKYU_OPERATOR,
        },
        "BusroutePattern": {
            "@type": "odpt:BusroutePattern",
            "odpt:operator": TOKYU_OPERATOR,
        },
        "BusstopPole": {
            "@type": "odpt:BusstopPole",
            "odpt:operator": TOKYU_OPERATOR,
        },
        "Calendar": {
            "@type": "odpt:Calendar",
        },
    }

    for name, params in STATIC_ENDPOINTS.items():
        # キャッシュチェック（force_refresh=False の場合）
        if not force_refresh:
            cached = load_cache_or_none(RAW_DIR, name)
            if cached is not None:
                continue  # キャッシュ有効 → スキップ

        # API取得
        print(f"[fetch] {name} を取得中...")
        odpt_type = params.pop("@type")
        data = fetch_endpoint(api_key, odpt_type, extra_params=params)
        save_cache(data, RAW_DIR, name)
        time.sleep(1)  # API過負荷防止（1秒待機）


def fetch_realtime_bus():
    """
    リアルタイム車両位置データを取得する（動的データ）。
    dct:valid が切れるたびに再取得が必要。
    研究本体でリアルタイムデータを使う場合のみ呼び出す。
    """
    api_key = get_api_key()

    # キャッシュチェック（動的データは短時間でも有効期限確認）
    cached = load_cache_or_none(RAW_DIR, "Bus")
    if cached is not None:
        return cached

    data = fetch_endpoint(api_key, "odpt:Bus", {
        "odpt:operator": TOKYU_OPERATOR,
    })
    save_cache(data, RAW_DIR, "Bus")
    return data


if __name__ == "__main__":
    print("=== ODPT 静的データ取得 ===")
    fetch_all_static()
    print("完了。次に odpt/normalize.py を実行してDB化してください。")
```

## 3. odpt/normalize.py の完全実装

報告書 3.2節（フラット化）・3.3節（SQLite）・3.5節（Parquet）を完全に実装する。

```python
# odpt/normalize.py
"""
data/raw/ の生JSON-LDを正規化し、SQLite と Parquet に変換する。

【テーブル構成（報告書 3.3節 推奨設計）】
  busstop_pole       : 停留所マスタ（owl:sameAs → 主キー）
  busroute_pattern   : 系統パターンヘッダ
  busroute_stop      : 系統内停留所順序（BusroutePatternのネスト展開）
  calendar           : カレンダーマスタ
  bus_timetable      : 時刻表ヘッダ
  bus_timetable_item : 時刻表明細（busTimetableObjectのネスト展開）
  _data_version      : データ更新バージョン管理

【設計原則（報告書 3.2節）】
  - owl:sameAs の値を TEXT PRIMARY KEY とする
  - ネスト配列は別テーブルに分解し、親IDで外部参照
  - 名前空間プレフィックス（odpt:, dc:, geo:等）を列名から除去
  - 結合キーには CREATE INDEX を設定（10〜100倍高速化）
  - PRAGMA journal_mode=WAL を有効化（並行アクセス改善）
"""

import json
import sqlite3
import pandas as pd
from pathlib import Path
from typing import Optional

from odpt.cache import find_latest_cache, record_db_version

RAW_DIR = "data/raw"
DB_PATH = "data/processed/trips.db"
PARQUET_DIR = "data/processed"


# ============================================================
# スキーマ定義
# ============================================================

DDL_STATEMENTS = [
    # WALモード（最初に設定）
    "PRAGMA journal_mode=WAL",

    # 停留所マスタ
    """
    CREATE TABLE IF NOT EXISTS busstop_pole (
        id          TEXT PRIMARY KEY,   -- owl:sameAs
        title       TEXT,               -- dc:title
        title_kana  TEXT,               -- odpt:kana
        operator    TEXT,               -- odpt:operator
        lat         REAL,               -- geo:lat
        lon         REAL                -- geo:long
    )
    """,

    # 系統パターンヘッダ
    """
    CREATE TABLE IF NOT EXISTS busroute_pattern (
        id          TEXT PRIMARY KEY,   -- owl:sameAs
        operator    TEXT,               -- odpt:operator
        busroute    TEXT,               -- odpt:busroute
        direction   TEXT,               -- odpt:direction (inbound/outbound)
        dc_date     TEXT                -- dc:date
    )
    """,

    # 系統内停留所順序（BusroutePatternのodpt:busStopPoleOrder展開）
    """
    CREATE TABLE IF NOT EXISTS busroute_stop (
        pattern_id  TEXT,               -- busroute_pattern.id への外部参照
        seq         INTEGER,            -- 停留所順序番号
        stop_id     TEXT,               -- odpt:busstopPole
        distance_m  REAL                -- odpt:note（距離情報がある場合）
    )
    """,

    # カレンダーマスタ
    """
    CREATE TABLE IF NOT EXISTS calendar (
        id          TEXT PRIMARY KEY,   -- owl:sameAs
        title       TEXT                -- dc:title (平日/土曜/日曜祝日)
    )
    """,

    # 時刻表ヘッダ
    """
    CREATE TABLE IF NOT EXISTS bus_timetable (
        id          TEXT PRIMARY KEY,   -- owl:sameAs
        operator    TEXT,               -- odpt:operator
        busroute    TEXT,               -- odpt:busroute
        pattern     TEXT,               -- odpt:busroutePattern
        calendar    TEXT,               -- odpt:calendar
        dc_date     TEXT                -- dc:date
    )
    """,

    # 時刻表明細（odpt:busTimetableObject のネスト展開）
    """
    CREATE TABLE IF NOT EXISTS bus_timetable_item (
        timetable_id    TEXT,           -- bus_timetable.id への外部参照
        seq             INTEGER,        -- 停留所順序番号
        stop_id         TEXT,           -- odpt:busstopPole
        departure_time  TEXT,           -- odpt:departureTime (HH:MM形式)
        arrival_time    TEXT,           -- odpt:arrivalTime
        is_non_step     INTEGER         -- odpt:isNonStep (0/1)
    )
    """,

    # バージョン管理テーブル
    """
    CREATE TABLE IF NOT EXISTS _data_version (
        endpoint    TEXT PRIMARY KEY,
        source_file TEXT,
        updated_at  TEXT
    )
    """,

    # インデックス（検索・JOIN高速化）
    "CREATE INDEX IF NOT EXISTS idx_timetable_route   ON bus_timetable(busroute)",
    "CREATE INDEX IF NOT EXISTS idx_timetable_pattern ON bus_timetable(pattern)",
    "CREATE INDEX IF NOT EXISTS idx_timetable_cal     ON bus_timetable(calendar)",
    "CREATE INDEX IF NOT EXISTS idx_item_tid          ON bus_timetable_item(timetable_id)",
    "CREATE INDEX IF NOT EXISTS idx_item_stop         ON bus_timetable_item(stop_id)",
    "CREATE INDEX IF NOT EXISTS idx_rtstop_pattern    ON busroute_stop(pattern_id)",
    "CREATE INDEX IF NOT EXISTS idx_stop_operator     ON busstop_pole(operator)",
]


# ============================================================
# フラット化関数（報告書 3.2節）
# ============================================================

def flatten_busstop_pole(obj: dict) -> dict:
    """odpt:BusstopPole → busstop_pole テーブル行"""
    return {
        "id":         obj.get("owl:sameAs"),
        "title":      obj.get("dc:title"),
        "title_kana": obj.get("odpt:kana"),
        "operator":   obj.get("odpt:operator"),
        "lat":        obj.get("geo:lat"),
        "lon":        obj.get("geo:long"),
    }


def flatten_busroute_pattern(obj: dict) -> tuple[dict, list[dict]]:
    """
    odpt:BusroutePattern → (busroute_pattern行, busroute_stop行リスト)
    ネスト配列 odpt:busStopPoleOrder を展開して別テーブルに分解する。
    """
    header = {
        "id":        obj.get("owl:sameAs"),
        "operator":  obj.get("odpt:operator"),
        "busroute":  obj.get("odpt:busroute"),
        "direction": obj.get("odpt:direction"),
        "dc_date":   obj.get("dc:date"),
    }
    pattern_id = header["id"]
    stops = []
    for seq, stop_item in enumerate(obj.get("odpt:busStopPoleOrder", [])):
        stops.append({
            "pattern_id": pattern_id,
            "seq":        seq,
            "stop_id":    stop_item.get("odpt:busstopPole"),
            "distance_m": stop_item.get("odpt:note"),  # 距離情報（存在すれば）
        })
    return header, stops


def flatten_bus_timetable(obj: dict) -> tuple[dict, list[dict]]:
    """
    odpt:BusTimetable → (bus_timetable行, bus_timetable_item行リスト)
    深いネスト配列 odpt:busTimetableObject を展開して別テーブルに分解する。
    【報告書 2.1節】odpt:busTimetableObject は時刻・停留所の配列で
    1件のJSONオブジェクトが数百行に及ぶケースがあるため、必ず分解する。
    """
    header = {
        "id":       obj.get("owl:sameAs"),
        "operator": obj.get("odpt:operator"),
        "busroute": obj.get("odpt:busroute"),
        "pattern":  obj.get("odpt:busroutePattern"),
        "calendar": obj.get("odpt:calendar"),
        "dc_date":  obj.get("dc:date"),
    }
    timetable_id = header["id"]
    items = []
    for seq, item in enumerate(obj.get("odpt:busTimetableObject", [])):
        items.append({
            "timetable_id":   timetable_id,
            "seq":            seq,
            "stop_id":        item.get("odpt:busstopPole"),
            "departure_time": item.get("odpt:departureTime"),
            "arrival_time":   item.get("odpt:arrivalTime"),
            "is_non_step":    int(item.get("odpt:isNonStep", False)),
        })
    return header, items


def flatten_calendar(obj: dict) -> dict:
    """odpt:Calendar → calendar テーブル行"""
    return {
        "id":    obj.get("owl:sameAs"),
        "title": obj.get("dc:title"),
    }


# ============================================================
# DB構築メイン関数
# ============================================================

def build_sqlite(
    raw_dir: str = RAW_DIR,
    db_path: str = DB_PATH,
    target_routes: Optional[list[str]] = None,
):
    """
    data/raw/ の全JSONを読み込んでSQLiteに正規化ロードする。
    target_routes が指定された場合、その路線のみに絞り込む（推奨）。

    【報告書 6.1節 Step 1〜3 の実装】
    """
    Path(db_path).parent.mkdir(parents=True, exist_ok=True)
    con = sqlite3.connect(db_path)

    # スキーマ作成（DDL）
    for stmt in DDL_STATEMENTS:
        con.execute(stmt)
    con.commit()
    print(f"[normalize] スキーマ作成完了: {db_path}")

    # --- BusstopPole ---
    cache = find_latest_cache(raw_dir, "BusstopPole")
    if cache:
        with open(cache, encoding="utf-8") as f:
            data = json.load(f)
        rows = [flatten_busstop_pole(obj) for obj in data]
        df = pd.DataFrame(rows).drop_duplicates("id").dropna(subset=["id"])
        df.to_sql("busstop_pole", con, if_exists="replace", index=False)
        print(f"[normalize] busstop_pole: {len(df)}件")
        record_db_version(db_path, "BusstopPole", str(cache))

    # --- BusroutePattern ---
    cache = find_latest_cache(raw_dir, "BusroutePattern")
    if cache:
        with open(cache, encoding="utf-8") as f:
            data = json.load(f)
        # 対象路線フィルタ
        if target_routes:
            data = [obj for obj in data if obj.get("odpt:busroute") in target_routes]
        headers, all_stops = [], []
        for obj in data:
            h, stops = flatten_busroute_pattern(obj)
            headers.append(h)
            all_stops.extend(stops)
        pd.DataFrame(headers).drop_duplicates("id").dropna(subset=["id"]) \
            .to_sql("busroute_pattern", con, if_exists="replace", index=False)
        pd.DataFrame(all_stops) \
            .to_sql("busroute_stop", con, if_exists="replace", index=False)
        print(f"[normalize] busroute_pattern: {len(headers)}件, busroute_stop: {len(all_stops)}件")
        record_db_version(db_path, "BusroutePattern", str(cache))

    # --- Calendar ---
    cache = find_latest_cache(raw_dir, "Calendar")
    if cache:
        with open(cache, encoding="utf-8") as f:
            data = json.load(f)
        rows = [flatten_calendar(obj) for obj in data]
        pd.DataFrame(rows).drop_duplicates("id").dropna(subset=["id"]) \
            .to_sql("calendar", con, if_exists="replace", index=False)
        print(f"[normalize] calendar: {len(rows)}件")

    # --- BusTimetable（最もデータ量が多い・ネスト深い） ---
    cache = find_latest_cache(raw_dir, "BusTimetable")
    if cache:
        with open(cache, encoding="utf-8") as f:
            data = json.load(f)
        # 対象路線フィルタ（重要：全事業者データは膨大になるため必ず絞り込む）
        if target_routes:
            data = [obj for obj in data if obj.get("odpt:busroute") in target_routes]
        headers, all_items = [], []
        for obj in data:
            h, items = flatten_bus_timetable(obj)
            headers.append(h)
            all_items.extend(items)
        pd.DataFrame(headers).drop_duplicates("id").dropna(subset=["id"]) \
            .to_sql("bus_timetable", con, if_exists="replace", index=False)
        pd.DataFrame(all_items) \
            .to_sql("bus_timetable_item", con, if_exists="replace", index=False)
        print(f"[normalize] bus_timetable: {len(headers)}件, bus_timetable_item: {len(all_items)}件")
        record_db_version(db_path, "BusTimetable", str(cache))

    # インデックス再作成（to_sql後に実行）
    for stmt in [s for s in DDL_STATEMENTS if s.strip().startswith("CREATE INDEX")]:
        con.execute(stmt)
    con.commit()
    con.close()
    print(f"[normalize] SQLite構築完了: {db_path}")


# ============================================================
# Parquet変換（報告書 3.5節）
# ============================================================

def export_trips_parquet(
    db_path: str = DB_PATH,
    out_dir: str = PARQUET_DIR,
):
    """
    SQLiteの正規化テーブルをJOINしてParquetに出力する。
    研究本体での高速読み込み用。
    圧縮形式: snappy（高速圧縮・解凍。ファイルサイズは生JSONの10〜20%）

    【報告書 3.5節より】
    - 生JSON比でファイルサイズ 10〜20%
    - pandas.read_parquet() で列選択取得が可能
    - DuckDBから直接クエリ可能
    """
    Path(out_dir).mkdir(parents=True, exist_ok=True)
    con = sqlite3.connect(db_path)

    # トリップ一覧（時刻表 × 停留所 × 時刻の結合）
    df_trips = pd.read_sql_query("""
        SELECT
            t.busroute,
            t.pattern,
            t.calendar,
            t.id               AS timetable_id,
            i.seq,
            i.stop_id,
            i.departure_time,
            i.arrival_time,
            i.is_non_step
        FROM bus_timetable t
        JOIN bus_timetable_item i ON t.id = i.timetable_id
        ORDER BY t.busroute, t.pattern, t.calendar, i.seq
    """, con)

    # 系統パターン（停留所順序）
    df_patterns = pd.read_sql_query("""
        SELECT
            rp.id   AS pattern_id,
            rp.busroute,
            rp.direction,
            rs.seq,
            rs.stop_id,
            rs.distance_m,
            bp.title AS stop_title,
            bp.lat,
            bp.lon
        FROM busroute_pattern rp
        JOIN busroute_stop rs ON rp.id = rs.pattern_id
        LEFT JOIN busstop_pole bp ON rs.stop_id = bp.id
        ORDER BY rp.busroute, rp.direction, rs.seq
    """, con)

    con.close()

    # Parquet出力（snappy圧縮）
    trips_out = Path(out_dir) / "trips.parquet"
    patterns_out = Path(out_dir) / "route_patterns.parquet"

    df_trips.to_parquet(trips_out, compression="snappy", index=False)
    df_patterns.to_parquet(patterns_out, compression="snappy", index=False)

    print(f"[parquet] trips:    {trips_out}  ({len(df_trips):,}行)")
    print(f"[parquet] patterns: {patterns_out}  ({len(df_patterns):,}行)")


if __name__ == "__main__":

    print("=== ODPT データ正規化・DB化 ===")
    build_sqlite(target_routes=MEGURO_ROUTES)
```

## 4. odpt/id_index.py の完全実装（owl:sameAs辞書解決）

報告書 3.7節「ID参照の事前解決」を完全実装する。

```python
# odpt/id_index.py
"""
ODPTオブジェクトのIDインデックスを構築するモジュール。

【報告書 3.7節より】
ODPTデータは owl:sameAs を外部参照IDとして使うリンクドデータ構造をとる。
研究コード内で都度SQL/API参照するのは効率が悪く、起動時に
全マスタデータをDictとしてメモリにロードして O(1) で解決する。

使用例:
    from odpt.id_index import StopIndex
    idx = StopIndex(db_path="data/processed/trips.db")
    stop = idx["odpt.BusstopPole:TokyuBus.Meguro.1"]
    print(stop["title"], stop["lat"], stop["lon"])
"""

import sqlite3
from typing import Optional


class StopIndex:
    """
    バス停IDインデックス（owl:sameAs → 停留所情報辞書）。
    O(1) でIDから停留所情報を引ける。
    """
    def __init__(self, db_path: str = "data/processed/trips.db"):
        con = sqlite3.connect(db_path)
        con.row_factory = sqlite3.Row
        rows = con.execute(
            "SELECT id, title, operator, lat, lon FROM busstop_pole"
        ).fetchall()
        con.close()
        # owl:sameAs をキーとした辞書を構築（O(1)参照）
        self._index = {row["id"]: dict(row) for row in rows}
        print(f"[id_index] StopIndex: {len(self._index)}件ロード")

    def __getitem__(self, stop_id: str) -> Optional[dict]:
        return self._index.get(stop_id)

    def get(self, stop_id: str, default=None):
        return self._index.get(stop_id, default)

    def __contains__(self, stop_id: str) -> bool:
        return stop_id in self._index

    def all_ids(self) -> list[str]:
        return list(self._index.keys())


class RouteIndex:
    """
    路線IDインデックス（owl:sameAs → 系統パターン情報）。
    """
    def __init__(self, db_path: str = "data/processed/trips.db"):
        con = sqlite3.connect(db_path)
        con.row_factory = sqlite3.Row
        patterns = con.execute(
            "SELECT id, busroute, direction FROM busroute_pattern"
        ).fetchall()
        # 系統ごとの停留所リスト（順序付き）
        stops_by_pattern = {}
        for pat in patterns:
            stops = con.execute("""
                SELECT rs.seq, rs.stop_id, bp.title, bp.lat, bp.lon
                FROM busroute_stop rs
                LEFT JOIN busstop_pole bp ON rs.stop_id = bp.id
                WHERE rs.pattern_id = ?
                ORDER BY rs.seq
            """, (pat["id"],)).fetchall()
            stops_by_pattern[pat["id"]] = [dict(s) for s in stops]
        con.close()

        self._index = {}
        for pat in patterns:
            self._index[pat["id"]] = {
                **dict(pat),
                "stops": stops_by_pattern.get(pat["id"], []),
            }
        print(f"[id_index] RouteIndex: {len(self._index)}件ロード")

    def __getitem__(self, pattern_id: str) -> Optional[dict]:
        return self._index.get(pattern_id)

    def get_stops(self, pattern_id: str) -> list[dict]:
        entry = self._index.get(pattern_id)
        return entry["stops"] if entry else []
```

## 5. odpt/to_trips.py：ODPTデータ → 研究入力CSVへの変換

研究本体が受け取る `trips_meguro.csv` を生成するスクリプト。

```python
# odpt/to_trips.py
"""
SQLiteのODPTデータを研究本体が受け取るトリップリストCSVに変換する。

出力: data/input/trips_meguro.csv
形式:
    trip_id, route, direction, pattern_id, calendar,
    dep_stop_id, dep_stop_title, dep_time,
    arr_stop_id, arr_stop_title, arr_time,
    seq, distance_km
"""

import sqlite3
import pandas as pd
from pathlib import Path


def generate_trips_csv(
    db_path: str = "data/processed/trips.db",
    out_path: str = "data/input/trips_meguro.csv",
    target_routes: list[str] = None,
    calendar_filter: str = None,  # 例: "odpt.Calendar:Weekday"
):
    """
    時刻表明細と停留所情報を結合し、連続する停留所ペアを
    1トリップ（区間）として展開するCSVを生成する。
    """
    Path(out_path).parent.mkdir(parents=True, exist_ok=True)
    con = sqlite3.connect(db_path)

    # クエリ：連続停留所ペアをトリップとして展開
    query = """
        WITH numbered AS (
            SELECT
                t.id           AS timetable_id,
                t.busroute,
                t.pattern,
                t.calendar,
                i.seq,
                i.stop_id,
                i.departure_time,
                bp.title        AS stop_title,
                bp.lat, bp.lon,
                -- 次の停留所情報（LEAD関数）
                LEAD(i.stop_id)         OVER w AS next_stop_id,
                LEAD(i.departure_time)  OVER w AS next_dep_time,
                LEAD(bp.title)          OVER w AS next_stop_title,
                LEAD(bp.lat)            OVER w AS next_lat,
                LEAD(bp.lon)            OVER w AS next_lon
            FROM bus_timetable t
            JOIN bus_timetable_item i ON t.id = i.timetable_id
            LEFT JOIN busstop_pole bp ON i.stop_id = bp.id
            WHERE 1=1
              {route_filter}
              {calendar_filter}
            WINDOW w AS (PARTITION BY t.id ORDER BY i.seq)
        )
        SELECT
            timetable_id || '_' || seq   AS trip_id,
            busroute                      AS route,
            pattern                       AS pattern_id,
            calendar,
            stop_id                       AS dep_stop_id,
            stop_title                    AS dep_stop_title,
            departure_time                AS dep_time,
            next_stop_id                  AS arr_stop_id,
            next_stop_title               AS arr_stop_title,
            next_dep_time                 AS arr_time,
            lat                           AS dep_lat,
            lon                           AS dep_lon,
            next_lat                      AS arr_lat,
            next_lon                      AS arr_lon
        FROM numbered
        WHERE next_stop_id IS NOT NULL  -- 最終停留所の行を除外
        ORDER BY route, pattern_id, calendar, dep_time
    """.format(
        route_filter=f"AND t.busroute IN ({','.join(['?']*len(target_routes))})"
                     if target_routes else "",
        calendar_filter=f"AND t.calendar = ?" if calendar_filter else "",
    )

    params = []
    if target_routes:
        params.extend(target_routes)
    if calendar_filter:
        params.append(calendar_filter)

    df = pd.read_sql_query(query, con, params=params if params else None)
    con.close()

    df.to_csv(out_path, index=False, encoding="utf-8-sig")
    print(f"[to_trips] トリップCSV生成: {out_path} ({len(df):,}行)")
    return df


if __name__ == "__main__":
    from models.constants import MEGURO_ROUTES

    generate_trips_csv(
        target_routes=MEGURO_ROUTES,
        calendar_filter="odpt.Calendar:Weekday",  # 平日ダイヤのみ
    )
```

## 6. DuckDBによる高速クエリ（報告書 3.4節）

データ量が500MB以上になった場合、またはJSONファイルを直接クエリしたい場合に使う。

```python
# odpt/duckdb_query.py
"""
DuckDB を使ったODPTデータの高速クエリユーティリティ。

【報告書 3.4節より】
- read_json_auto() でJSONを直接SQLクエリ可能（変換不要）
- 複数CPUコアを活用した並列クエリ実行
- Parquet出力・入力が高速

用途：
  1. 生JSONの中身を素早く探索・確認したい時
  2. 集計クエリ（路線別件数、停留所別運行本数等）
  3. Parquetへの直接エクスポート
"""

import duckdb
import pandas as pd


def query_json_direct(json_path: str, sql: str) -> pd.DataFrame:
    """
    JSONファイルを直接SQLクエリする（変換不要）。

    例:
        df = query_json_direct(
            "data/raw/bustimetable_20250301.json",
            'SELECT "odpt:busroute", COUNT(*) AS cnt FROM data GROUP BY 1 ORDER BY 2 DESC'
        )
    """
    return duckdb.sql(
        sql.replace("FROM data", f"FROM read_json_auto('{json_path}')")
    ).df()


def inspect_json_schema(json_path: str):
    """
    JSONファイルの列名・型を確認する（設計時に使用）。
    """
    result = duckdb.sql(f"""
        DESCRIBE SELECT * FROM read_json_auto('{json_path}') LIMIT 1
    """).df()
    print(result.to_string())


def count_by_operator(json_path: str) -> pd.DataFrame:
    """事業者別件数集計（データ確認用）"""
    return duckdb.sql(f"""
        SELECT "odpt:operator" AS operator, COUNT(*) AS cnt
        FROM read_json_auto('{json_path}')
        GROUP BY "odpt:operator"
        ORDER BY cnt DESC
    """).df()


def count_by_route(json_path: str) -> pd.DataFrame:
    """路線別時刻表件数集計"""
    return duckdb.sql(f"""
        SELECT "odpt:busroute" AS busroute, COUNT(*) AS cnt
        FROM read_json_auto('{json_path}')
        GROUP BY "odpt:busroute"
        ORDER BY cnt DESC
    """).df()


def json_to_parquet_direct(json_path: str, out_path: str):
    """
    JSONファイルをParquetに直接変換（pandas不要・高速）。
    【報告書 3.5節】ファイルサイズが元JSONの10〜20%になる。
    """
    duckdb.sql(f"""
        COPY (SELECT * FROM read_json_auto('{json_path}'))
        TO '{out_path}' (FORMAT PARQUET, COMPRESSION SNAPPY)
    """)
    print(f"[duckdb] {json_path} → {out_path}")
```
