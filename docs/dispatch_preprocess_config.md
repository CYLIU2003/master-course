# dispatch_preprocess config reference

This note explains how `dispatch_preprocess` in config JSON controls
`travel_connections` rebuilding in `src/data_loader.py`.

## Purpose

- Keep timetable-first feasibility as the source of truth when needed.
- Decide whether to trust existing `travel_connection.csv` or rebuild connections.
- Select rebuild source: dispatch graph or `build_inputs` deadhead arcs.

## Location

Add this block at the top level of config (same level as `mode`, `paths`, `solver`).

```json
{
  "dispatch_preprocess": {
    "enabled": true,
    "connection_source": "dispatch_graph",
    "rebuild_when_missing": true,
    "force_rebuild_travel_connections": false,
    "default_turnaround_min": 10,
    "turnaround_rules_csv": "data/operations/turnaround_rules.csv",
    "deadhead_rules_csv": "data/operations/deadhead_rules.csv",
    "replace_tasks_from_build_inputs": true
  }
}
```

## Keys

- `enabled` (bool, default `true`)
  - Master switch. If `false`, no preprocess rebuild is executed.

- `connection_source` (string, default `"dispatch_graph"`)
  - `"dispatch_graph"`: rebuild from `src.dispatch` feasibility graph.
  - `"build_inputs"`: rebuild from `src.pipeline.build_inputs` deadhead arcs.
  - `"none"`: no rebuild (keep loaded `travel_connection_csv` as-is).

- `rebuild_when_missing` (bool, default `true`)
  - Rebuild only when `travel_connections` is empty after CSV loading.

- `force_rebuild_travel_connections` (bool, default `false`)
  - Always rebuild regardless of existing `travel_connection_csv` contents.

- `default_turnaround_min` (int, default `10`)
  - Fallback turnaround minutes for stops without explicit rule.
  - Used by `dispatch_graph` source.

- `turnaround_rules_csv` (string, optional)
  - Optional override path for turnaround rule CSV.
  - Used by `dispatch_graph` source.

- `deadhead_rules_csv` (string, optional)
  - Optional override path for deadhead rule CSV.
  - Used by `dispatch_graph` source.

- `replace_tasks_from_build_inputs` (bool, default `true`)
  - If `true` and source is `build_inputs`, replace `data.tasks` with trips
    generated by `build_inputs` before rebuilding connections.

## Recommended presets

### Keep case CSV as authority (stable reproduction)

```json
{
  "dispatch_preprocess": {
    "enabled": true,
    "connection_source": "dispatch_graph",
    "rebuild_when_missing": false,
    "force_rebuild_travel_connections": false
  }
}
```

### Rebuild automatically when `travel_connection.csv` is missing

```json
{
  "dispatch_preprocess": {
    "enabled": true,
    "connection_source": "dispatch_graph",
    "rebuild_when_missing": true,
    "force_rebuild_travel_connections": false
  }
}
```

### Compare with `build_inputs` arc pipeline

```json
{
  "dispatch_preprocess": {
    "enabled": true,
    "connection_source": "build_inputs",
    "rebuild_when_missing": true,
    "force_rebuild_travel_connections": true,
    "replace_tasks_from_build_inputs": true
  }
}
```

## Runtime log

When preprocess runs, `src/pipeline/solve.py` prints a dispatch line like:

```text
[dispatch] source=build_inputs, trips=..., edges=..., connections=..., vehicle_types=[...]
```
